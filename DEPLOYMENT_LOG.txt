WHY DEPLOY TO RENDER BEFORE CREATING THE SHOPIFY APP?
================================================================================

YES — deploy/host on Render FIRST. Reasons:

1. Shopify OAuth requires a real, public HTTPS callback URL.
   localhost URLs do not work for production OAuth (only ngrok works locally).

2. In Part 3 (Create app), Shopify asks for an "App URL" — that is your
   FRONTEND URL. You cannot know this URL until something is deployed.

3. The redirect URL must match BACKEND_PUBLIC_URL exactly:
   {BACKEND_PUBLIC_URL}/api/auth/callback

4. After Render deploy you get URLs like:
   Frontend: https://skincare-frontend.onrender.com
   Backend:  https://skincare-api.onrender.com

5. You can add custom domains later (app.yourdomain.com, api.yourdomain.com)
   and update Shopify settings once — no need to guess URLs upfront.

================================================================================
HOW TO GET THE FRONTEND URL (AND API URL)
================================================================================

OPTION A — RENDER DEFAULT URLS (easiest, use this first)
---------------------------------------------------------
After deploying via render.yaml, open Render Dashboard -> each service -> top
of page shows "URL".

  skincare-frontend service URL  = FRONTEND_PUBLIC_URL
                                   = SHOPIFY_APP_URL
                                   = App URL in Shopify Partner Dashboard
                                   Example: https://skincare-frontend.onrender.com

  skincare-api service URL       = BACKEND_PUBLIC_URL
                                   = NEXT_PUBLIC_API_URL
                                   Example: https://skincare-api.onrender.com

  Shopify redirect URL           = https://skincare-api.onrender.com/api/auth/callback

OPTION B — CUSTOM DOMAIN (later, optional)
------------------------------------------
In Render Dashboard -> service -> Settings -> Custom Domains:
  app.yourdomain.com   -> skincare-frontend
  api.yourdomain.com   -> skincare-api

Then update ALL of these to match:
  FRONTEND_PUBLIC_URL=https://app.yourdomain.com
  BACKEND_PUBLIC_URL=https://api.yourdomain.com
  NEXT_PUBLIC_API_URL=https://api.yourdomain.com
  CORS_ORIGINS=https://app.yourdomain.com
  SHOPIFY_APP_URL=https://app.yourdomain.com
  Shopify App URL: https://app.yourdomain.com
  Shopify Redirect: https://api.yourdomain.com/api/auth/callback

OPTION C — LOCAL DEV ONLY (not recommended for first Shopify setup)
--------------------------------------------------------------------
Use ngrok on port 8000 for backend callback only.
Frontend can stay on http://localhost:3000.
More complex; use Render first instead.

OAUTH FLOW REFERENCE (how URLs connect)
================================================================================

  Merchant clicks Connect (Settings page)
       |
       v
  GET {BACKEND_PUBLIC_URL}/api/auth/start?shop=store.myshopify.com
       |
       v
  Shopify OAuth screen (merchant approves)
       |
       v
  Shopify calls {BACKEND_PUBLIC_URL}/api/auth/callback?code=...&shop=...
       |
       v
  Backend saves access token + syncs catalog
       |
       v
  Redirect to {FRONTEND_PUBLIC_URL}/dashboard/settings?connected=1
